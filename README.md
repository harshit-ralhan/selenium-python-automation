# Selenium Python Automation

## Project Purpose

This project demonstrates UI automation testing using Selenium with Python and Pytest. It is designed to showcase framework design, reporting, and CI/CD integration for portfolio and interview preparation.

## Tech Stack

- Python 3.x
- Selenium 4
- Pytest
- Pytest-HTML (for reporting)
- GitHub Actions (CI/CD)

## Environment Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/harshit-ralhan/selenium-python-automation.git
   cd selenium-python-automation
   ```
2. Upgrade pip (recommended)

   ```bash
   python -m pip install --upgrade pip
   ```

3. Install all dependencies from requirements.txt

   ```bash
   python install -r requirements.txt
   ```

4. (Optional) Verify installation

   ```bash
   pip list
   ```

5. Install pytest-xdist for parallel execution of tests

```bash
python -m pip install pytest-xdist
```

## Running Tests Locally

You can run tests across different browsres and data sources in different combinations as per your choice.

```bash
pytest -v --browser=firefox --data-source=csv
pytest -v --browser=edge --data-source=excel
pytest -v --browser=chrome --data-source=json

```

## Parallel Execution

Run tests in parallel using pytest-xdist:

```bash
pytest -n 4
pytest -n 3 -v --browser=edge --data-source=csv
```

## Reports & Artifacts

Each test run produces three types of artifacts:

- **HTML Report**
  Generated by `pytest-html`, saved under:
  `reports/<browser>_<data_source>/report.html`

- **Screenshots (on failure)**
  Captured automatically when a test fails, saved under:
  `reports/<browser>_<data_source>/screenshots/`

  Here’s an example test report screenshot:

![Test Report Screenshot](reports/sample_report.png)

- **Logs**
  Timestamped log files for each run, saved under:
  `logs/<browser>_<data_source>/`

## Flaky Test Handling

- Implemented retry logic using `pytest-rerunfailures`
- Demonstrated with a simulated flaky test (`tests/test_flaky.py`)
- Ensures pipeline stability against intermittent UI failures

### How It Works

- Failing tests are retried automatically with:
  ```bash
  pytest -v --reruns 2 --reruns-delay 5 --html=reports/report.html --self-contained-html
  ```

## CI/CD Integration

This project uses **GitHub Actions** for continuous integration and delivery.
Every push or pull request to the `main` branch automatically triggers the test pipeline.

### Matrix Strategy

The workflow runs tests across:

- **Browsers**: Chrome, Firefox, Edge
- **Data sources**: JSON, CSV, Excel

This creates a **3 × 3 matrix = 9 jobs** executed in parallel, ensuring full coverage across all combinations.

### Artifacts

Each job uploads its own artifacts with clear names:

- `selenium-test-artifacts-chrome-json`
- `selenium-test-artifacts-firefox-csv`
- `selenium-test-artifacts-edge-excel`

Artifacts include:

- HTML report (`reports/<browser>_<data_source>/report.html`)
- Screenshots (`reports/<browser>_<data_source>/screenshots/`)
- Logs (`logs/<browser>_<data_source>/`)

This framework includes several recruiter‑friendly improvements:

- **Clickable Screenshots in Reports**
  Failed tests automatically embed screenshot links in the HTML report, making debugging faster and more professional.

- **Retry Logic for Flaky Tests**
  Integrated with `pytest-rerunfailures` so transient failures are retried automatically, improving CI/CD stability.

- **Build Badge**
  A GitHub Actions badge at the top of the README shows the current build status at a glance, demonstrating CI/CD integration.

## Project Highlights
